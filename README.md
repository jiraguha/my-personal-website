A spec-driven personal website that treats content as code. Blog posts, project showcases, conference talks, and short notes — all authored in markdown, server-rendered, and deployed as static HTML. No database. No CMS. Git is the source of truth.

## Quickstart

```bash
bun install
bun run dev        # → localhost:5173
```

No Docker needed for dev — just Bun.

## Features

### Content system

Four content types live as markdown files with YAML frontmatter. Drop a `.md` file in the right folder and it appears on the site — no config, no registration.

| Type | Path | What it is |
|------|------|-----------|
| **Blog** | `src/content/posts/blog/` | Long-form technical articles with AI-generated cover images |
| **Project** | `src/content/posts/project/` | Portfolio pieces with links, demos, and case studies |
| **Talk** | `src/content/posts/talk/` | Full Reveal.js slide decks authored in markdown — speaker notes, code highlighting, embedded Mermaid diagrams |
| **Short** | `src/content/posts/short/` | Quick tips, TILs, and thoughts — distinct violet-bordered cards in the grid |

Content is validated at the boundary with Zod schemas. Drafts and future-dated posts are visible in dev but excluded from production builds.

### AI-generated cover images

Every post gets a cover image generated by **Gemini**, styled with a unified cyberpunk HUD aesthetic — dark navy backgrounds, cyan and red accents, circuit-trace patterns, monospace text. The design is controlled by a single `da.md` file, so you can change the entire visual identity in one place.

Frontmatter fields give per-post control: custom keywords, composition hints, text density (`none` / `minimal` / `moderate` / `heavy`), and reproducible seeds. Each run outputs a display image (1200x675) and an OG-optimized variant (1200x630).

```bash
bun run covers           # generate missing covers
bun run cover -- <slug>  # generate for a single post
```

### Server-side rendering & SEO

Vike handles SSR and static prerendering. Every page ships complete HTML — no empty divs waiting for JavaScript.

- **Open Graph tags** on every page — category-aware OG images for rich social previews
- **JSON-LD structured data** — `Person` schema on home, `Article` schema on posts
- **Auto-generated sitemap** and `robots.txt` at build time
- **Canonical URLs** for all pages
- **Twitter Cards** with large image support

```bash
bun run test:seo    # run SEO audit
```

### Fuzzy search

Client-side search powered by Fuse.js. Press `/` or click the search icon to start typing.

Searches across titles (weight 0.4), summaries (0.3), and tags (0.3) with a 0.35 threshold — forgiving enough for typos, precise enough to be useful. Results filter in real-time with 150ms debounce and respect the active category filter.

### Trending tags

The home page highlights the top 8 trending tags, scored by a weighted algorithm:

```
score = frequency × 0.4 + recency × 0.6
recency = avg(e^(-0.01 × days_since_post))   # ~69-day half-life
```

Tags are visually tiered: top 2 get violet glow, 3-5 get indigo, the rest are muted. On mobile, they scroll horizontally with a fade hint.

### Reveal.js presentations

Talks are full-viewport slide decks rendered from markdown. Horizontal slides are separated by `---`, vertical sub-slides by `----`, and speaker notes start with `Note:`.

Features: code highlighting, embedded Mermaid diagrams with light/dark themes, a "back to site" overlay that fades after 3 seconds, and full Reveal.js keyboard controls. Posts can also link to external slide decks instead.

### Mermaid diagrams

Fenced `mermaid` code blocks in any post are rendered client-side with a custom theme matching the site palette. Invalid syntax shows a styled error card instead of breaking the page. Each diagram gets a unique ID to avoid Mermaid collision bugs.

### Dark mode

Theme preference is detected from localStorage via an inline script in `<head>` — no flash of wrong theme on load. Toggle persists across sessions.

### Dynamic favicon

Generated from a single letter defined in `profile.json`. Outputs SVG + PNG at 32, 192, and 512px. Cached by hash — only regenerates when the letter changes.

## Project structure

```
src/
  content/posts/          Markdown content (blog, project, talk, short)
  content/profile.json    Site identity (name, role, bio, socials)
  shared/schemas/         Zod schemas — the API ↔ UI contract
  ui/                     React frontend (Vite + Tailwind 4)
  api/                    Bun API server
test/
  lib/                    Vitest unit tests
  e2e/                    Playwright E2E tests
  content/                Isolated test fixtures
specs/                    Feature specs (SDD)
da.md                     Design aesthetic for cover generation
```

## How it's built

This project uses **spec-driven development**. Every feature starts as a numbered spec in [specs/](specs/), goes through failing tests, implementation, and verification. 15 specs completed.

Two modes:
- **Full** — spec, failing tests, implementation, verification. Nothing ships without passing tests.
- **Prototype** — skip tests, move fast. Prototype code gets promoted (add tests, verify) or deleted. It never stays as-is.
